Implementing Accessibility

Assumptions

  • Accessibility is completed by an accessibility expert

  • Accessibility is tested at the end of development by the expert

  • Issues are given to the developers to be fixed with the guidance of the expert

Reality

  • Testing is needed when design decisions are made, not code

  • Earlier input provides accessibility training across all roles in the development lifecycle

Input at the Design Stage

  • Define issues

  • Do not provide solutions
